主要讲解 yaml 的基本语法

1 yaml 简介#

1.1 基本语法#

  • 大小写敏感
  • 使用缩进表示层级关系
  • 缩进时不允许使用 Tab 键，只允许使用空格。
  • 缩进的空格数目不重要，只要相同层级的元素左侧对齐即可

对于某种语言的库读取 yaml 文件配置时可能大小写并不敏感，这个根据具体的库有关 ,比如 go 语言中的 cobra 就不敏感。

1. 2 注释#

# 表示区块高度

# 表示注释，从这个字符一直到行尾，都会被解析器忽略。

1.3 支持的数据结构#

  • 对象：键值对的集合，又称为映射（mapping）/ 哈希（hashes） / 字典（dictionary）
  • 数组：一组按次序排列的值，又称为序列（sequence） / 列表（list）
  • 纯量（scalars）：单个的、不可再分的值

2 数据结构#

2.1 对象#

使用冒号表示

animal: pets

2.2 数组#

子成员是一个数组，则可以在该项下面缩进一个空格。

languages:
 - Ruby
 - Perl
 - Python

2.3 纯量#

纯量是最基本的、不可再分的值

# 数值类型
number: 12.30  
# 布尔值用 true 和 false 表示。
isSet: true
# null用~表示。
parent: ~

字符串 字符串默认不使用引号表示。

str: 这是一行字符串

如果字符串之中包含空格或特殊字符，需要放在引号之中

{ str: '内容: 字符串' }

双引号和单引号都可以使用，不会进行转义

s1: '内容\n字符串'
s2: "内容\n字符串"

单引号之中如果还有单引号，必须连续使用两个单引号转义。

str: '内容''字符串'

字符串可以写成多行，从第二行开始，必须有一个单空格缩进。换行符会被转为空格

str: 今天
 天气
 真好

多行字符串可以使用|保留换行符，也可以使用>折叠换行。

str: |
 今天
 天气
 真好

结果为

今天 天气 真好

str: >
 今天
 天气
 真好

结果为

今天天气真好

3 引用#

锚点 & 和别名 *，可以用来引用。

defaults: &defaults
  adapter:  postgres
  host:     localhost

development:
  database: myapp_development
  <<: *defaults

test:
  database: myapp_test
  <<: *defaults

等同于下面代码

defaults:
  adapter:  postgres
  host:     localhost

development:
  database: myapp_development
  adapter:  postgres
  host:     localhost

test:
  database: myapp_test
  adapter:  postgres
  host:     localhost
